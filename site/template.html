Per §12 of CLAUDE.md, every project keeps a running log of substantive turns as a Claude-native artifact, mirrored to session-log.html. New entries are added newest-first. Here is a template entry, in that format, covering how to open a Claude Code session against a project once CLAUDE.md is in place.

Entry — template session-log.html
Question

Once a project has a CLAUDE.md in place, how do I actually start a Claude Code session against it?

Answer
VS Code extension. Open the project folder in VS Code, then either install "Claude Code" from the Extensions Marketplace directly, or run claude in VS Code's integrated terminal — it offers to install the companion extension automatically. Once installed, the extension reads CLAUDE.md from the project root as context for every session opened against that folder; no separate loading step is needed. Sessions are opened and driven from the extension's panel and the Command Palette rather than a specific fixed keybinding, since that detail varies by extension version — search the palette for "Claude Code" if the panel isn't already visible.
JetBrains plugin. The same "Claude Code" plugin is published to the JetBrains Marketplace for IntelliJ-family IDEs, with the same CLAUDE.md-on-open behavior.
CLI. Run claude from a terminal inside the project directory. This is editor-independent and is what the IDE extensions wrap under the hood.
Desktop or web app. The desktop app (Mac/Windows) and the web app at claude.ai/code cover the same workflow for projects pushed to a connected GitHub repository, for cases where a local editor isn't the point of entry.
